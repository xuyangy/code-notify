Feature release that keeps Claude completion alerts accurate while delegated
work is still running, with Antigravity and idle-notification fixes on top of
`v2026.07.3`.

## Highlights

### Claude delegated-work awareness

- Claude `Stop` events now defer task-complete alerts and native idle reminders
  while the event snapshot still reports a running subagent, teammate, cloud
  session, or remote agent.
- Session-scoped delegated-work state is retired when Claude reports
  `SubagentStop` or `TeammateIdle`, and expires automatically after a fail-open
  timeout so stale state cannot suppress alerts indefinitely.
- Delegated-work retirement hooks are installed even when their optional
  user-facing event alerts are disabled.
- `cn off` now recognizes older managed Claude hook sets, ensuring upgraded
  installations can always remove their hooks cleanly.
- The same behavior is available in the Unix and Windows hook runtimes.

## Notification fixes

- Prevented queued Antigravity permission prompts from rapidly switching a
  tmux window between its running spinner and waiting badge.
- Idle reminders now keep the originating tool's subtitle instead of falling
  back to a generic label.
