Patch release with tmux reliability fixes on top of `v2026.07.0`.

## Highlights

### No more duplicated notifications from tmux timers

- Fixed Codex/Antigravity idle reminders and settle completions firing two or
  three times at once. Racing hook processes could each arm the internal tmux
  sweep/poll timers, and every stray timer chain then survived and re-armed
  forever, duplicating each notification it raced on.
- Timer registrations now carry an ownership token: a fired timer runs only
  while it still owns the chain, so accidental duplicates collapse on their
  first firing. Ownership is claimed before the timer is armed, and a failed
  arm rolls back atomically, so a stalled or raced scheduler can neither
  duplicate the chain nor wedge it.

### Running-indicator accuracy

- The running indicator no longer falsely resumes while a permission or input
  dialog is still on screen: pane animation behind a visible dialog (for
  example, a backgrounded command's ticking timer) is no longer mistaken for
  an answer.

### Cosmetics

- The `TaskCompleted` agent-team event now badges with 🏅 instead of reusing
  the task-complete 🟢.
