Maintenance release that keeps tmux activity indicators and Claude completion
alerts accurate across concurrent and delegated work, on top of `v2026.07.4`.

## Tmux queued-prompt handling

- A prompt submitted while the previous turn is stopping now keeps the tmux
  running indicator active for the queued successor turn.
- Prompt submission and terminal teardown use a recoverable per-window lock,
  preventing racing Stop events from clearing a successor's fresh running
  marker or overwriting its badge.
- Queued-prompt hints are consumed by the matching Stop only and expire with
  the existing running-state timeout.

## Claude delegated-work completion

- `TeammateIdle` now leaves a session-scoped tombstone so a parked teammate's
  stale `running` registry entry cannot suppress every later completion alert.
- Active subagents, cloud sessions, and remote agents remain authoritative and
  continue to defer completion alerts while they are running.
- The teammate retirement behavior is available in both Unix and Windows hook
  runtimes.
